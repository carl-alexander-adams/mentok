Check Environment Module

Overview

The Check Environment Module provides the build system with the ability to run environment checks on the build environment.

Macros and Controls

The Check Environment Module currently does not have any control macros

Targets

  • checkenv_info
  • checkenv_man
  • checkenv_ckospkg

checkenv_info

checkenv_info Overview

Simple target to display information about the macro settings of all global macros used by the check environment module of the build system. The checkenv_info target will be built during the "info" phase of a build, and can be built in isolation by requesting the checkenv_info target.

checkenv_man

checkenv_man Overview

Simple target to display usage information and help pertaining to the check envirionment module of the build system. The checkenv_man target will be built during the "man" phase of a build, and can be built in isolation by requesting the checkenv_man target.

checkenv_ckospkg

checkenv_ckospkg Overview

Simple target to trigger CKOSPKG_TARGETS pattern targets in isolation.

Targets

  • CKOSPKG_TARGETS

    CKOSPKG_TARGETS Overview

    CKOSPKG_TARGETS is the pattern target for checking that a given OS package is installed. If the requested OS package is not found, a build failure will result. Package check targets can specify a package name (which defaults to the target name), version, release, and architecture. When version, release, or arch are not specified, any found version satisifies the target. Wild cards may be used on OSs that support this (such as YUM based systems)

    CKOSPKG_TARGETS Overview

    EXE_TARGETS Options

    <ckospkg target>_NAME
    Override the default package name to ckeck for. The default is the name of the target itself.

    <ckospkg target>_VERSION
    Check for a particular version of the installed package. The default is to accept any found version of the package.

    <ckospkg target>_RELEASE
    Check for a particular release of the installed package. The default is to accept any found release of the package.

    <ckospkg target>_ARCH
    Check for a particular architecture of the installed package. The default is to accept any found architecture of the package. NOTE: while architecure names provoded by the base module of build3 may be useful, this architecure must be specified per the underlaying operating system's expectations.

    Example Makefile

    
# The source root of this Component
COMPONENT_ROOT=../..

# The root of the build system
BS_ROOT=$(COMPONENT_ROOT)/build3

# Boostrap Build system definitions.
include $(BS_ROOT)/defines.mk

# check for a particular version of postgres-devel with a package 
# name override, and any version of openssl with all defaults.

CKOSPKG_TARGETS=\
    postgres-devel \
	openssl

postgres-server_NAME=postgresql-devel
postgres-server_VERSION=8.1.18
postgres-server_RELEASE=e15_4.1
postgres-server_ARCH=x86_64

# Finally, include the rules to make the build system go.
include $(BS_ROOT)/rules.mk
