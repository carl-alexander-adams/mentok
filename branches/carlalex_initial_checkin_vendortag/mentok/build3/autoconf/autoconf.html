Build System Autoconf Module

Overview

The Autoconf build system module provides the ability to run autoconf to generate config.h files for source code that still requires this.

This module is provided to support legacy code. New code should not depend on autoconf.

CONFIGH_TARGETS

CONFIGH_TARGETS Overview

CONFIGH_TARGETS is the pattern target to create autoconf style "config.h" files. The resulting source code files from build stamp targets are placed in the BS_ARCH_TARGET_DIR.

CONFIGH_TARGETS will be built during the "pretarget" phase of a build. CONFIGH_TARGETS may be built in isolation by requesting the "autoconf_configh" target.

CONFIGH_TARGETS Options

autoconf_configh

autoconf_configh Overview

Simple target to trigger the building of CONFIGH_TARGETS pattern targets independent of the main build "pretarget" phase.

autoconf_info

autoconf_info Overview

Simple target to display information about the macro settings of all global macros used by the autoconf module of the build system. The autoconf_info target will be built during the "info" phase of a build, and can be built in isolation by requesting the autoconf_info target.

autoconf_man

autoconf_man Overview

Simple target to display usage information and help pertaining to the autoconf module of the build system. The autoconf_man target will be built during the "man" phase of a build, and can be built in isolation by requesting the autoconf_man target.

Example Makefile


# The source root of this component
COMPONENT_ROOT=../..

# The root of the build system
BS_ROOT=$(COMPONENT_ROOT)/../build3

# Bootstrap Build system definitions.
include $(BS_ROOT)/defines.mk

CONFIGH_TARGETS=configure-foo.h


# Finally, include the rules to make the build system go.
include $(BS_ROOT)/rules.mk
